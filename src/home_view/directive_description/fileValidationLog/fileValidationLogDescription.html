fileTabs API Documentation

<file-tabs> is a component that is used for the display of errors, warnings and other such details about uploaded files that need to be shown to the user before they proceed further.

To start you would need to specify the data attribute.
The file name is the key to an object. Each object will have arrays of the details that the users wishes to display. Here we are using only two details warnings and errors but the user can display summary as well. It will also be in the same manner. 

sample_data = {
    "file 1": {
        "errors": ["error1", "error2", "error3", "error4", "error5", "error6"],
        "warnings": []
    },
    "file 2": {
        "errors": ["error1", "error2", "error3", "error4"],
        "warnings": ["warning1", "warning2", "warning3", "warning4"]
    },
}

Note:- All the objects in the JSON object should contain all the arrays. If the array does not have any entries associated to it for that particular file, then the list can be left empty, but it needs to be defined. For references please have a look at the above example.

Usage

Basic Example

<file-tabs data = "sample_data" listcolors='{errors:"#EA4436",warnings:"#C57D00"}' labelsize='1.9em' textsize='1em' download-icon="block" page-size=5></file-tabs>

Attributes

Parameter Type Description
data object

model for file validation logs.

listcolors object

color corresponding to each list in the given data The keys will be the same as the names of the arrays in each object of the json data and the colors need to be input in the hex/string format. Default value is black for all the accordions.

labelsize number

the font size of the accordion label.It is defined like 20px.It will take values just like the font-size attribute of css.

textsize number

font size of the body of each accordion.It is defined like 20px.It will take values just like the font-size attribute of css

defines the number of items per page when applying pagination.Default is 20
